# Menus MD3 Specs

Source: <https://m3.material.io/components/menus/specs>
Collected: 2026-03-11

## Summary

- Four token sets are exposed in the primary token viewer: baseline, common, vibrant, and standard.
- Expressive menus use two color mappings: standard is surface-based and vibrant is tertiary-based.
- The main page exposes baseline measurements as text, but the expressive vertical-menu measurements are image-only in the rendered page.
- Baseline rows inside the shared token viewer include many entries marked as deprecated.

## Implementation Notes

- Prefer the expressive vertical-menu token sets for new work and keep the baseline set only for compatibility paths.
- Treat standard and vibrant mappings as separate style families; their token namespaces and color roles differ materially.
- Preserve support for hover, focus, pressed, disabled, and focus-indicator states because each token set exposes state-specific overrides.
- Where the token viewer exposes shape or elevation only as previews, keep those values marked visual-only unless another MD3 source provides concrete numeric values.
